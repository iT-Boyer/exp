Patch Notes

Release (2007-01-19)

Released: January 19, 2007

  • Performance Improvements.
  • A bug fix for: Submitting only relevant nodes.

Release (2006-11-02)

Released: November 02, 2006

  • Updated labels to handle label text being determined by a ref.
  • Added support for submit validation for constraint and required.
  • Added "Loading" image, while page is loading, initializing, and rendering.
  • Updated input and select to trigger a DOMActivate event when the enter key is pressed within either the input or select controls.
  • Several bug fixes for: "xforms-insert" is actually dispatched on insert rather than "xforms-delete", I.E. 7 XML HTTP Request Object, select options in I.E., controls bound to non-existent nodes are treated as non-relevant.

Release (2006-03-03)

Released: March 3, 2006

  • Added back dropdown-style select controls, which is now the default style. You can get radio buttons or check boxes with the appearance="full" option.
  • Added support for XML documents styled with an XSLT stylesheet that generates an XForms page.
  • Removed default styling for relevance (hidden) and control labels (small and above the control). Styling is now up to the page author. Note: relevance has no visual effect by default; form authors might want to add a CSS rule like .xforms-disabled { display: none; } to their pages.
  • Added element classes for each pseudo-class: xforms-invalid for :invalid, xforms-optional for :optional, etc.
  • Added workaround for Opera bug/quirk that was causing FormFaces to fail.
  • Several bug fixes for: deleting last focused row of a repeat, quotes in serialized attributes, bound labels inside triggers inside a repeat.

Release (2006-02-16)

Released: February 16, 2006

  • Fixed several bugs with submission.
  • Added debugging statements for figuring out why submissions fail.
  • Beefed up the code for handling a non-XHTML response.

Release (2006-01-31)

Released: January 31, 2006

  • Added replace="all" for submission
  • Fixed bug when running in Firefox 1.5 causing all the functions to stop working.
  • Fixed bug causing errors to not return properly.

Alpha (2005-11-21)

Released: November 21, 2005

  • Added the XPath function current() from the XSLT function library.
  • Fixed the index() function for repeats.
  • Added the dynamic xforms-repeat-index CSS class which is added to the current row of a repeat. See the test pages using repeats for examples of its use.

Alpha (2005-10-07)

Released: October 7, 2005

  • Added support for the itemset element, which allows you to dynamically populate select controls with choices taken from the instance data. This allows for AJAX-like on-the-fly loading of data.
  • The ev:handler attribute can now give the name of a JavaScript function to call when an event is caught.
  • Fixed bug causing enormous slowdown on pages with large, inline instance data.

Alpha (2005-08-29)

Released: August 29, 2005

  • Large rewrite of the XForms→HTML rendering algorithm, resulting in significant speed-ups for model rebuilds, and opening up many avenues for future optimization.
  • Added support for the XML events <ev:listener/> element and the ev:handler attribute.

Alpha (2005-08-01)

Released: August 1, 2005

  • External instance data supported.
  • Full submission process implemented, and debugging messages removed. replace="none" and replace="instance" are supported.
  • More actions and events implemented.

Alpha (2005-07-05)

Released: July 5, 2005

  • index() function for repeats.
  • Many more actions and events implemented.
  • Serialization as application/x-www-form-urlencoded; get and urlencoded-post submission methods.
  • Revamped example pages.
  • Compressed JavaScript to one file (size 77K), files uncompressed (size 280K) still available.
